# Module 01: Syntax Comparison

Welcome to the first module of JavaScript to Java learning path! In this module, we'll explore the fundamental syntax differences between JavaScript and Java. Understanding these differences is crucial for JavaScript developers transitioning to Java.

## Learning Objectives

By the end of this module, you will be able to:
- Understand the key syntax differences between JavaScript and Java
- Write basic Java code with proper syntax
- Compare variable declarations, function definitions, and control structures
- Apply Java coding conventions and best practices

## Key Differences Overview

### 1. Variable Declaration

JavaScript uses dynamic typing with `let`, `const`, and `var`, while Java uses static typing with explicit type declarations.

<UniversalEditor title="Variable Declaration Comparison" compare={true}>
```javascript !! js
// JavaScript - Dynamic typing
let name = "John";
const age = 25;
var city = "New York";

// Types are inferred automatically
let isStudent = true;
let score = 95.5;
```

```java !! java
// Java - Static typing
String name = "John";
final int age = 25;
String city = "New York";

// Types must be explicitly declared
boolean isStudent = true;
double score = 95.5;
```
</UniversalEditor>

### 2. Function Definition

JavaScript functions are flexible and can be defined in multiple ways, while Java methods are more structured and must be part of a class.

<UniversalEditor title="Function Definition Comparison" compare={true}>
```javascript !! js
// JavaScript - Multiple ways to define functions
// Function declaration
function greet(name) {
    return "Hello, " + name + "!";
}

// Function expression
const greetArrow = (name) => {
    return "Hello, " + name + "!";
};

// Arrow function with implicit return
const greetSimple = name => "Hello, " + name + "!";

// Function call
console.log(greet("Alice"));
console.log(greetArrow("Bob"));
console.log(greetSimple("Charlie"));
```

```java !! java
// Java - Methods must be part of a class
public class Greeter {
    // Method with explicit return type
    public String greet(String name) {
        return "Hello, " + name + "!";
    }
    
    // Method with void return type
    public void printGreeting(String name) {
        System.out.println("Hello, " + name + "!");
    }
    
    // Static method (can be called without creating an instance)
    public static String staticGreet(String name) {
        return "Hello, " + name + "!";
    }
}

// Usage
public class Main {
    public static void main(String[] args) {
        Greeter greeter = new Greeter();
        System.out.println(greeter.greet("Alice"));
        greeter.printGreeting("Bob");
        System.out.println(Greeter.staticGreet("Charlie"));
    }
}
```
</UniversalEditor>

### 3. Control Structures

Both languages have similar control structures, but Java requires explicit boolean expressions and semicolons.

<UniversalEditor title="Control Structures Comparison" compare={true}>
```javascript !! js
// JavaScript - Control structures
let age = 18;
let score = 85;

// If-else statement
if (age >= 18) {
    console.log("Adult");
} else {
    console.log("Minor");
}

// Switch statement
switch (score) {
    case 90:
        console.log("Excellent");
        break;
    case 80:
        console.log("Good");
        break;
    default:
        console.log("Keep trying");
}

// For loop
for (let i = 0; i < 5; i++) {
    console.log("Count: " + i);
}

// While loop
let count = 0;
while (count < 3) {
    console.log("While count: " + count);
    count++;
}

// For...of loop (JavaScript specific)
const fruits = ["apple", "banana", "orange"];
for (const fruit of fruits) {
    console.log(fruit);
}
```

```java !! java
// Java - Control structures
public class ControlStructures {
    public static void main(String[] args) {
        int age = 18;
        int score = 85;
        
        // If-else statement
        if (age >= 18) {
            System.out.println("Adult");
        } else {
            System.out.println("Minor");
        }
        
        // Switch statement (Java 14+ enhanced switch)
        switch (score) {
            case 90:
                System.out.println("Excellent");
                break;
            case 80:
                System.out.println("Good");
                break;
            default:
                System.out.println("Keep trying");
        }
        
        // For loop
        for (int i = 0; i < 5; i++) {
            System.out.println("Count: " + i);
        }
        
        // While loop
        int count = 0;
        while (count < 3) {
            System.out.println("While count: " + count);
            count++;
        }
        
        // Enhanced for loop (Java equivalent of for...of)
        String[] fruits = {"apple", "banana", "orange"};
        for (String fruit : fruits) {
            System.out.println(fruit);
        }
    }
}
```
</UniversalEditor>

### 4. Operators and Expressions

Java and JavaScript share most operators, but Java has stricter type checking.

<UniversalEditor title="Operators and Expressions Comparison" compare={true}>
```javascript !! js
// JavaScript - Operators
let a = 10;
let b = 3;
let c = "5";

// Arithmetic operators
console.log(a + b);    // 13
console.log(a - b);    // 7
console.log(a * b);    // 30
console.log(a / b);    // 3.333...
console.log(a % b);    // 1

// String concatenation
console.log(a + c);    // "105" (number + string = string)

// Comparison operators
console.log(a == c);   // true (loose equality)
console.log(a === c);  // false (strict equality)

// Logical operators
console.log(true && false);  // false
console.log(true || false);  // true
console.log(!true);          // false

// Increment/Decrement
console.log(a++);  // 10 (post-increment)
console.log(++a);  // 12 (pre-increment)
```

```java !! java
// Java - Operators
public class Operators {
    public static void main(String[] args) {
        int a = 10;
        int b = 3;
        String c = "5";
        
        // Arithmetic operators
        System.out.println(a + b);    // 13
        System.out.println(a - b);    // 7
        System.out.println(a * b);    // 30
        System.out.println(a / b);    // 3 (integer division)
        System.out.println(a % b);    // 1
        
        // String concatenation
        System.out.println(a + c);    // "105" (number + string = string)
        
        // Comparison operators
        // System.out.println(a == c);   // Compilation error (can't compare int and String)
        System.out.println(a == Integer.parseInt(c));  // true
        
        // Logical operators
        System.out.println(true && false);  // false
        System.out.println(true || false);  // true
        System.out.println(!true);          // false
        
        // Increment/Decrement
        System.out.println(a++);  // 10 (post-increment)
        System.out.println(++a);  // 12 (pre-increment)
        
        // Type casting
        double result = (double) a / b;  // 4.0 (explicit casting for floating-point division)
        System.out.println(result);
    }
}
```
</UniversalEditor>

## Java Coding Conventions

### 1. Naming Conventions

<UniversalEditor title="Naming Conventions" compare={true}>
```javascript !! js
// JavaScript - camelCase for variables and functions
let userName = "john_doe";
const MAX_RETRY_COUNT = 3;

function calculateTotalPrice() {
    // function body
}

class UserAccount {
    constructor() {
        // constructor body
    }
    
    getUserInfo() {
        // method body
    }
}
```

```java !! java
// Java - camelCase for variables and methods, PascalCase for classes
public class UserAccount {
    // Constants: UPPER_SNAKE_CASE
    public static final int MAX_RETRY_COUNT = 3;
    
    // Instance variables: camelCase
    private String userName = "john_doe";
    
    // Methods: camelCase
    public void calculateTotalPrice() {
        // method body
    }
    
    public String getUserInfo() {
        // method body
        return userName;
    }
    
    // Constructor: same name as class
    public UserAccount() {
        // constructor body
    }
}
```
</UniversalEditor>

### 2. Code Structure

<UniversalEditor title="Code Structure Comparison" compare={true}>
```javascript !! js
// JavaScript - Flexible structure
// Can have functions and variables at top level
const PI = 3.14159;

function calculateArea(radius) {
    return PI * radius * radius;
}

// Can also use classes
class Circle {
    constructor(radius) {
        this.radius = radius;
    }
    
    getArea() {
        return PI * this.radius * this.radius;
    }
}

// Usage
console.log(calculateArea(5));
const circle = new Circle(5);
console.log(circle.getArea());
```

```java !! java
// Java - Everything must be in a class
public class CircleCalculator {
    // Constants
    public static final double PI = 3.14159;
    
    // Static method
    public static double calculateArea(double radius) {
        return PI * radius * radius;
    }
    
    // Main method - entry point
    public static void main(String[] args) {
        System.out.println(calculateArea(5));
        
        Circle circle = new Circle(5);
        System.out.println(circle.getArea());
    }
}

// Separate class definition
class Circle {
    private double radius;
    
    public Circle(double radius) {
        this.radius = radius;
    }
    
    public double getArea() {
        return CircleCalculator.PI * this.radius * this.radius;
    }
}
```
</UniversalEditor>

## Common Syntax Pitfalls

### 1. Semicolons and Braces

<UniversalEditor title="Semicolons and Braces" compare={true}>
```javascript !! js
// JavaScript - Semicolons are optional but recommended
function greet(name) {
    return "Hello, " + name;  // semicolon optional
}

// Braces are required for multi-line blocks
if (name) {
    console.log("Name exists");
    console.log("Processing...");
}

// Single-line blocks can omit braces (not recommended)
if (name) console.log("Name exists");
```

```java !! java
// Java - Semicolons are required
public class SyntaxExample {
    public String greet(String name) {
        return "Hello, " + name;  // semicolon required
    }
    
    public void processName(String name) {
        // Braces are always required
        if (name != null) {
            System.out.println("Name exists");
            System.out.println("Processing...");
        }
        
        // Single-line blocks also need braces
        if (name != null) {
            System.out.println("Name exists");
        }
    }
}
```
</UniversalEditor>

### 2. Type Safety

<UniversalEditor title="Type Safety Comparison" compare={true}>
```javascript !! js
// JavaScript - Dynamic typing allows flexibility
let value = 42;
console.log(typeof value);  // "number"

value = "Hello";
console.log(typeof value);  // "string"

value = [1, 2, 3];
console.log(typeof value);  // "object"

// Type coercion
console.log(5 + "5");       // "55" (string)
console.log(5 - "5");       // 0 (number)
console.log(5 == "5");      // true (loose equality)
console.log(5 === "5");     // false (strict equality)
```

```java !! java
// Java - Static typing prevents type errors
public class TypeSafety {
    public static void main(String[] args) {
        int value = 42;
        System.out.println("Type: int");
        
        // value = "Hello";  // Compilation error!
        String stringValue = "Hello";
        System.out.println("Type: String");
        
        // No type coercion in arithmetic
        // System.out.println(5 + "5");  // Compilation error!
        System.out.println(5 + Integer.parseInt("5"));  // 10
        
        // No loose equality
        // System.out.println(5 == "5");  // Compilation error!
        System.out.println(5 == Integer.parseInt("5"));  // true
    }
}
```
</UniversalEditor>

## Summary

In this module, we've covered the fundamental syntax differences between JavaScript and Java:

### Key Takeaways:
1. **Static vs Dynamic Typing**: Java requires explicit type declarations, while JavaScript infers types
2. **Class-based Structure**: Java requires everything to be in a class, while JavaScript is more flexible
3. **Method Definitions**: Java methods must have explicit return types and be part of a class
4. **Type Safety**: Java prevents type-related errors at compile time
5. **Coding Conventions**: Java has stricter naming and formatting conventions
