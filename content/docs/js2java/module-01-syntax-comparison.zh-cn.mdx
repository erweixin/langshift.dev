# 模块 01: 语法对比

欢迎来到 JavaScript 到 Java 学习路径的第一个模块！在本模块中，我们将探索 JavaScript 和 Java 之间的基础语法差异。理解这些差异对于从 JavaScript 转向 Java 的开发者来说至关重要。

## 学习目标

通过本模块的学习，你将能够：
- 理解 JavaScript 和 Java 之间的关键语法差异
- 使用正确的语法编写基础 Java 代码
- 对比变量声明、函数定义和控制结构
- 应用 Java 编码规范和最佳实践

## 主要差异概述

### 1. 变量声明

JavaScript 使用动态类型和 `let`、`const`、`var`，而 Java 使用静态类型和显式类型声明。

<UniversalEditor title="变量声明对比" compare={true}>
```javascript !! js
// JavaScript - 动态类型
let name = "John";
const age = 25;
var city = "New York";

// 类型自动推断
let isStudent = true;
let score = 95.5;
```

```java !! java
// Java - 静态类型
String name = "John";
final int age = 25;
String city = "New York";

// 必须显式声明类型
boolean isStudent = true;
double score = 95.5;
```
</UniversalEditor>

### 2. 函数定义

JavaScript 函数灵活，可以用多种方式定义，而 Java 方法更加结构化，必须是类的一部分。

<UniversalEditor title="函数定义对比" compare={true}>
```javascript !! js
// JavaScript - 多种定义函数的方式
// 函数声明
function greet(name) {
    return "Hello, " + name + "!";
}

// 函数表达式
const greetArrow = (name) => {
    return "Hello, " + name + "!";
};

// 箭头函数，隐式返回
const greetSimple = name => "Hello, " + name + "!";

// 函数调用
console.log(greet("Alice"));
console.log(greetArrow("Bob"));
console.log(greetSimple("Charlie"));
```

```java !! java
// Java - 方法必须是类的一部分
public class Greeter {
    // 带显式返回类型的方法
    public String greet(String name) {
        return "Hello, " + name + "!";
    }
    
    // 带 void 返回类型的方法
    public void printGreeting(String name) {
        System.out.println("Hello, " + name + "!");
    }
    
    // 静态方法（无需创建实例即可调用）
    public static String staticGreet(String name) {
        return "Hello, " + name + "!";
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Greeter greeter = new Greeter();
        System.out.println(greeter.greet("Alice"));
        greeter.printGreeting("Bob");
        System.out.println(Greeter.staticGreet("Charlie"));
    }
}
```
</UniversalEditor>

### 3. 控制结构

两种语言都有相似的控制结构，但 Java 需要显式的布尔表达式和分号。

<UniversalEditor title="控制结构对比" compare={true}>
```javascript !! js
// JavaScript - 控制结构
let age = 18;
let score = 85;

// If-else 语句
if (age >= 18) {
    console.log("Adult");
} else {
    console.log("Minor");
}

// Switch 语句
switch (score) {
    case 90:
        console.log("Excellent");
        break;
    case 80:
        console.log("Good");
        break;
    default:
        console.log("Keep trying");
}

// For 循环
for (let i = 0; i < 5; i++) {
    console.log("Count: " + i);
}

// While 循环
let count = 0;
while (count < 3) {
    console.log("While count: " + count);
    count++;
}

// For...of 循环（JavaScript 特有）
const fruits = ["apple", "banana", "orange"];
for (const fruit of fruits) {
    console.log(fruit);
}
```

```java !! java
// Java - 控制结构
public class ControlStructures {
    public static void main(String[] args) {
        int age = 18;
        int score = 85;
        
        // If-else 语句
        if (age >= 18) {
            System.out.println("Adult");
        } else {
            System.out.println("Minor");
        }
        
        // Switch 语句（Java 14+ 增强 switch）
        switch (score) {
            case 90:
                System.out.println("Excellent");
                break;
            case 80:
                System.out.println("Good");
                break;
            default:
                System.out.println("Keep trying");
        }
        
        // For 循环
        for (int i = 0; i < 5; i++) {
            System.out.println("Count: " + i);
        }
        
        // While 循环
        int count = 0;
        while (count < 3) {
            System.out.println("While count: " + count);
            count++;
        }
        
        // 增强 for 循环（Java 的 for...of 等价物）
        String[] fruits = {"apple", "banana", "orange"};
        for (String fruit : fruits) {
            System.out.println(fruit);
        }
    }
}
```
</UniversalEditor>

### 4. 运算符和表达式

Java 和 JavaScript 共享大部分运算符，但 Java 有更严格的类型检查。

<UniversalEditor title="运算符和表达式对比" compare={true}>
```javascript !! js
// JavaScript - 运算符
let a = 10;
let b = 3;
let c = "5";

// 算术运算符
console.log(a + b);    // 13
console.log(a - b);    // 7
console.log(a * b);    // 30
console.log(a / b);    // 3.333...
console.log(a % b);    // 1

// 字符串连接
console.log(a + c);    // "105" (数字 + 字符串 = 字符串)

// 比较运算符
console.log(a == c);   // true (宽松相等)
console.log(a === c);  // false (严格相等)

// 逻辑运算符
console.log(true && false);  // false
console.log(true || false);  // true
console.log(!true);          // false

// 递增/递减
console.log(a++);  // 10 (后递增)
console.log(++a);  // 12 (前递增)
```

```java !! java
// Java - 运算符
public class Operators {
    public static void main(String[] args) {
        int a = 10;
        int b = 3;
        String c = "5";
        
        // 算术运算符
        System.out.println(a + b);    // 13
        System.out.println(a - b);    // 7
        System.out.println(a * b);    // 30
        System.out.println(a / b);    // 3 (整数除法)
        System.out.println(a % b);    // 1
        
        // 字符串连接
        System.out.println(a + c);    // "105" (数字 + 字符串 = 字符串)
        
        // 比较运算符
        // System.out.println(a == c);   // 编译错误（不能比较 int 和 String）
        System.out.println(a == Integer.parseInt(c));  // true
        
        // 逻辑运算符
        System.out.println(true && false);  // false
        System.out.println(true || false);  // true
        System.out.println(!true);          // false
        
        // 递增/递减
        System.out.println(a++);  // 10 (后递增)
        System.out.println(++a);  // 12 (前递增)
        
        // 类型转换
        double result = (double) a / b;  // 4.0 (显式转换进行浮点除法)
        System.out.println(result);
    }
}
```
</UniversalEditor>

## Java 编码规范

### 1. 命名规范

<UniversalEditor title="命名规范" compare={true}>
```javascript !! js
// JavaScript - 变量和函数使用 camelCase
let userName = "john_doe";
const MAX_RETRY_COUNT = 3;

function calculateTotalPrice() {
    // 函数体
}

class UserAccount {
    constructor() {
        // 构造函数体
    }
    
    getUserInfo() {
        // 方法体
    }
}
```

```java !! java
// Java - 变量和方法使用 camelCase，类使用 PascalCase
public class UserAccount {
    // 常量：UPPER_SNAKE_CASE
    public static final int MAX_RETRY_COUNT = 3;
    
    // 实例变量：camelCase
    private String userName = "john_doe";
    
    // 方法：camelCase
    public void calculateTotalPrice() {
        // 方法体
    }
    
    public String getUserInfo() {
        // 方法体
        return userName;
    }
    
    // 构造函数：与类同名
    public UserAccount() {
        // 构造函数体
    }
}
```
</UniversalEditor>

### 2. 代码结构

<UniversalEditor title="代码结构对比" compare={true}>
```javascript !! js
// JavaScript - 灵活的结构
// 可以在顶层有函数和变量
const PI = 3.14159;

function calculateArea(radius) {
    return PI * radius * radius;
}

// 也可以使用类
class Circle {
    constructor(radius) {
        this.radius = radius;
    }
    
    getArea() {
        return PI * this.radius * this.radius;
    }
}

// 使用
console.log(calculateArea(5));
const circle = new Circle(5);
console.log(circle.getArea());
```

```java !! java
// Java - 一切都必须在类中
public class CircleCalculator {
    // 常量
    public static final double PI = 3.14159;
    
    // 静态方法
    public static double calculateArea(double radius) {
        return PI * radius * radius;
    }
    
    // 主方法 - 入口点
    public static void main(String[] args) {
        System.out.println(calculateArea(5));
        
        Circle circle = new Circle(5);
        System.out.println(circle.getArea());
    }
}

// 单独的类定义
class Circle {
    private double radius;
    
    public Circle(double radius) {
        this.radius = radius;
    }
    
    public double getArea() {
        return CircleCalculator.PI * this.radius * this.radius;
    }
}
```
</UniversalEditor>

## 常见语法陷阱

### 1. 分号和花括号

<UniversalEditor title="分号和花括号" compare={true}>
```javascript !! js
// JavaScript - 分号可选但推荐使用
function greet(name) {
    return "Hello, " + name;  // 分号可选
}

// 多行块需要花括号
if (name) {
    console.log("Name exists");
    console.log("Processing...");
}

// 单行块可以省略花括号（不推荐）
if (name) console.log("Name exists");
```

```java !! java
// Java - 分号必需
public class SyntaxExample {
    public String greet(String name) {
        return "Hello, " + name;  // 分号必需
    }
    
    public void processName(String name) {
        // 总是需要花括号
        if (name != null) {
            System.out.println("Name exists");
            System.out.println("Processing...");
        }
        
        // 单行块也需要花括号
        if (name != null) {
            System.out.println("Name exists");
        }
    }
}
```
</UniversalEditor>

### 2. 类型安全

<UniversalEditor title="类型安全对比" compare={true}>
```javascript !! js
// JavaScript - 动态类型允许灵活性
let value = 42;
console.log(typeof value);  // "number"

value = "Hello";
console.log(typeof value);  // "string"

value = [1, 2, 3];
console.log(typeof value);  // "object"

// 类型强制转换
console.log(5 + "5");       // "55" (字符串)
console.log(5 - "5");       // 0 (数字)
console.log(5 == "5");      // true (宽松相等)
console.log(5 === "5");     // false (严格相等)
```

```java !! java
// Java - 静态类型防止类型错误
public class TypeSafety {
    public static void main(String[] args) {
        int value = 42;
        System.out.println("Type: int");
        
        // value = "Hello";  // 编译错误！
        String stringValue = "Hello";
        System.out.println("Type: String");
        
        // 算术运算中没有类型强制转换
        // System.out.println(5 + "5");  // 编译错误！
        System.out.println(5 + Integer.parseInt("5"));  // 10
        
        // 没有宽松相等
        // System.out.println(5 == "5");  // 编译错误！
        System.out.println(5 == Integer.parseInt("5"));  // true
    }
}
```
</UniversalEditor>

## 总结

在本模块中，我们涵盖了 JavaScript 和 Java 之间的基础语法差异：

### 关键要点：
1. **静态 vs 动态类型**: Java 需要显式类型声明，而 JavaScript 推断类型
2. **基于类的结构**: Java 要求一切都必须在类中，而 JavaScript 更灵活
3. **方法定义**: Java 方法必须有显式返回类型且必须是类的一部分
4. **类型安全**: Java 在编译时防止类型相关错误
5. **编码规范**: Java 有更严格的命名和格式规范
