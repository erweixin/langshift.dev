# 模組 01: 語法對比

歡迎來到 JavaScript 到 Java 學習路徑的第一個模組！在本模組中，我們將探索 JavaScript 和 Java 之間的基礎語法差異。理解這些差異對於從 JavaScript 轉向 Java 的開發者來說至關重要。

## 學習目標

通過本模組的學習，你將能夠：
- 理解 JavaScript 和 Java 之間的關鍵語法差異
- 使用正確的語法編寫基礎 Java 程式碼
- 對比變數宣告、函數定義和控制結構
- 應用 Java 編碼規範和最佳實踐

## 主要差異概述

### 1. 變數宣告

JavaScript 使用動態類型和 `let`、`const`、`var`，而 Java 使用靜態類型和顯式類型宣告。

<UniversalEditor title="變數宣告對比" compare={true}>
```javascript !! js
// JavaScript - 動態類型
let name = "John";
const age = 25;
var city = "New York";

// 類型自動推斷
let isStudent = true;
let score = 95.5;
```

```java !! java
// Java - 靜態類型
String name = "John";
final int age = 25;
String city = "New York";

// 必須顯式宣告類型
boolean isStudent = true;
double score = 95.5;
```
</UniversalEditor>

### 2. 函數定義

JavaScript 函數靈活，可以用多種方式定義，而 Java 方法更加結構化，必須是類的一部分。

<UniversalEditor title="函數定義對比" compare={true}>
```javascript !! js
// JavaScript - 多種定義函數的方式
// 函數宣告
function greet(name) {
    return "Hello, " + name + "!";
}

// 函數表達式
const greetArrow = (name) => {
    return "Hello, " + name + "!";
};

// 箭頭函數，隱式返回
const greetSimple = name => "Hello, " + name + "!";

// 函數呼叫
console.log(greet("Alice"));
console.log(greetArrow("Bob"));
console.log(greetSimple("Charlie"));
```

```java !! java
// Java - 方法必須是類的一部分
public class Greeter {
    // 帶顯式返回類型的方法
    public String greet(String name) {
        return "Hello, " + name + "!";
    }
    
    // 帶 void 返回類型的方法
    public void printGreeting(String name) {
        System.out.println("Hello, " + name + "!");
    }
    
    // 靜態方法（無需建立實例即可呼叫）
    public static String staticGreet(String name) {
        return "Hello, " + name + "!";
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Greeter greeter = new Greeter();
        System.out.println(greeter.greet("Alice"));
        greeter.printGreeting("Bob");
        System.out.println(Greeter.staticGreet("Charlie"));
    }
}
```
</UniversalEditor>

### 3. 控制結構

兩種語言都有相似的控制結構，但 Java 需要顯式的布林表達式和分號。

<UniversalEditor title="控制結構對比" compare={true}>
```javascript !! js
// JavaScript - 控制結構
let age = 18;
let score = 85;

// If-else 語句
if (age >= 18) {
    console.log("Adult");
} else {
    console.log("Minor");
}

// Switch 語句
switch (score) {
    case 90:
        console.log("Excellent");
        break;
    case 80:
        console.log("Good");
        break;
    default:
        console.log("Keep trying");
}

// For 迴圈
for (let i = 0; i < 5; i++) {
    console.log("Count: " + i);
}

// While 迴圈
let count = 0;
while (count < 3) {
    console.log("While count: " + count);
    count++;
}

// For...of 迴圈（JavaScript 特有）
const fruits = ["apple", "banana", "orange"];
for (const fruit of fruits) {
    console.log(fruit);
}
```

```java !! java
// Java - 控制結構
public class ControlStructures {
    public static void main(String[] args) {
        int age = 18;
        int score = 85;
        
        // If-else 語句
        if (age >= 18) {
            System.out.println("Adult");
        } else {
            System.out.println("Minor");
        }
        
        // Switch 語句（Java 14+ 增強 switch）
        switch (score) {
            case 90:
                System.out.println("Excellent");
                break;
            case 80:
                System.out.println("Good");
                break;
            default:
                System.out.println("Keep trying");
        }
        
        // For 迴圈
        for (int i = 0; i < 5; i++) {
            System.out.println("Count: " + i);
        }
        
        // While 迴圈
        int count = 0;
        while (count < 3) {
            System.out.println("While count: " + count);
            count++;
        }
        
        // 增強 for 迴圈（Java 的 for...of 等價物）
        String[] fruits = {"apple", "banana", "orange"};
        for (String fruit : fruits) {
            System.out.println(fruit);
        }
    }
}
```
</UniversalEditor>

### 4. 運算子和表達式

Java 和 JavaScript 共享大部分運算子，但 Java 有更嚴格的類型檢查。

<UniversalEditor title="運算子和表達式對比" compare={true}>
```javascript !! js
// JavaScript - 運算子
let a = 10;
let b = 3;
let c = "5";

// 算術運算子
console.log(a + b);    // 13
console.log(a - b);    // 7
console.log(a * b);    // 30
console.log(a / b);    // 3.333...
console.log(a % b);    // 1

// 字串連接
console.log(a + c);    // "105" (數字 + 字串 = 字串)

// 比較運算子
console.log(a == c);   // true (寬鬆相等)
console.log(a === c);  // false (嚴格相等)

// 邏輯運算子
console.log(true && false);  // false
console.log(true || false);  // true
console.log(!true);          // false

// 遞增/遞減
console.log(a++);  // 10 (後遞增)
console.log(++a);  // 12 (前遞增)
```

```java !! java
// Java - 運算子
public class Operators {
    public static void main(String[] args) {
        int a = 10;
        int b = 3;
        String c = "5";
        
        // 算術運算子
        System.out.println(a + b);    // 13
        System.out.println(a - b);    // 7
        System.out.println(a * b);    // 30
        System.out.println(a / b);    // 3 (整數除法)
        System.out.println(a % b);    // 1
        
        // 字串連接
        System.out.println(a + c);    // "105" (數字 + 字串 = 字串)
        
        // 比較運算子
        // System.out.println(a == c);   // 編譯錯誤（不能比較 int 和 String）
        System.out.println(a == Integer.parseInt(c));  // true
        
        // 邏輯運算子
        System.out.println(true && false);  // false
        System.out.println(true || false);  // true
        System.out.println(!true);          // false
        
        // 遞增/遞減
        System.out.println(a++);  // 10 (後遞增)
        System.out.println(++a);  // 12 (前遞增)
        
        // 類型轉換
        double result = (double) a / b;  // 4.0 (顯式轉換進行浮點除法)
        System.out.println(result);
    }
}
```
</UniversalEditor>

## Java 編碼規範

### 1. 命名規範

<UniversalEditor title="命名規範" compare={true}>
```javascript !! js
// JavaScript - 變數和函數使用 camelCase
let userName = "john_doe";
const MAX_RETRY_COUNT = 3;

function calculateTotalPrice() {
    // 函數體
}

class UserAccount {
    constructor() {
        // 建構函數體
    }
    
    getUserInfo() {
        // 方法體
    }
}
```

```java !! java
// Java - 變數和方法使用 camelCase，類使用 PascalCase
public class UserAccount {
    // 常數：UPPER_SNAKE_CASE
    public static final int MAX_RETRY_COUNT = 3;
    
    // 實例變數：camelCase
    private String userName = "john_doe";
    
    // 方法：camelCase
    public void calculateTotalPrice() {
        // 方法體
    }
    
    public String getUserInfo() {
        // 方法體
        return userName;
    }
    
    // 建構函數：與類同名
    public UserAccount() {
        // 建構函數體
    }
}
```
</UniversalEditor>

### 2. 程式碼結構

<UniversalEditor title="程式碼結構對比" compare={true}>
```javascript !! js
// JavaScript - 靈活的結構
// 可以在頂層有函數和變數
const PI = 3.14159;

function calculateArea(radius) {
    return PI * radius * radius;
}

// 也可以使用類
class Circle {
    constructor(radius) {
        this.radius = radius;
    }
    
    getArea() {
        return PI * this.radius * this.radius;
    }
}

// 使用
console.log(calculateArea(5));
const circle = new Circle(5);
console.log(circle.getArea());
```

```java !! java
// Java - 一切都必須在類中
public class CircleCalculator {
    // 常數
    public static final double PI = 3.14159;
    
    // 靜態方法
    public static double calculateArea(double radius) {
        return PI * radius * radius;
    }
    
    // 主方法 - 入口點
    public static void main(String[] args) {
        System.out.println(calculateArea(5));
        
        Circle circle = new Circle(5);
        System.out.println(circle.getArea());
    }
}

// 單獨的類定義
class Circle {
    private double radius;
    
    public Circle(double radius) {
        this.radius = radius;
    }
    
    public double getArea() {
        return CircleCalculator.PI * this.radius * this.radius;
    }
}
```
</UniversalEditor>

## 常見語法陷阱

### 1. 分號和花括號

<UniversalEditor title="分號和花括號" compare={true}>
```javascript !! js
// JavaScript - 分號可選但推薦使用
function greet(name) {
    return "Hello, " + name;  // 分號可選
}

// 多行塊需要花括號
if (name) {
    console.log("Name exists");
    console.log("Processing...");
}

// 單行塊可以省略花括號（不推薦）
if (name) console.log("Name exists");
```

```java !! java
// Java - 分號必需
public class SyntaxExample {
    public String greet(String name) {
        return "Hello, " + name;  // 分號必需
    }
    
    public void processName(String name) {
        // 總是需要花括號
        if (name != null) {
            System.out.println("Name exists");
            System.out.println("Processing...");
        }
        
        // 單行塊也需要花括號
        if (name != null) {
            System.out.println("Name exists");
        }
    }
}
```
</UniversalEditor>

### 2. 類型安全

<UniversalEditor title="類型安全對比" compare={true}>
```javascript !! js
// JavaScript - 動態類型允許靈活性
let value = 42;
console.log(typeof value);  // "number"

value = "Hello";
console.log(typeof value);  // "string"

value = [1, 2, 3];
console.log(typeof value);  // "object"

// 類型強制轉換
console.log(5 + "5");       // "55" (字串)
console.log(5 - "5");       // 0 (數字)
console.log(5 == "5");      // true (寬鬆相等)
console.log(5 === "5");     // false (嚴格相等)
```

```java !! java
// Java - 靜態類型防止類型錯誤
public class TypeSafety {
    public static void main(String[] args) {
        int value = 42;
        System.out.println("Type: int");
        
        // value = "Hello";  // 編譯錯誤！
        String stringValue = "Hello";
        System.out.println("Type: String");
        
        // 算術運算中沒有類型強制轉換
        // System.out.println(5 + "5");  // 編譯錯誤！
        System.out.println(5 + Integer.parseInt("5"));  // 10
        
        // 沒有寬鬆相等
        // System.out.println(5 == "5");  // 編譯錯誤！
        System.out.println(5 == Integer.parseInt("5"));  // true
    }
}
```
</UniversalEditor>

## 總結

在本模組中，我們涵蓋了 JavaScript 和 Java 之間的基礎語法差異：

### 關鍵要點：
1. **靜態 vs 動態類型**: Java 需要顯式類型宣告，而 JavaScript 推斷類型
2. **基於類的結構**: Java 要求一切都必須在類中，而 JavaScript 更靈活
3. **方法定義**: Java 方法必須有顯式返回類型且必須是類的一部分
4. **類型安全**: Java 在編譯時防止類型相關錯誤
5. **編碼規範**: Java 有更嚴格的命名和格式規範
