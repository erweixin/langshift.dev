## 變量和可變性

你注意到的第一個主要區別是 Rust 如何處理變量。

<UniversalEditor compare={true} title="變量聲明:Python vs Rust">
```python !! py
# Python:變量是引用,默認可變
name = "Alice"
age = 25
height = 5.6

# 可以隨時重新賦值
age = 26

# 甚至可以改變類型
age = "twenty-six"  # 這可以工作!
```

```rust !! rs
// Rust:變量是綁定,默認不可變
let name = "Alice";
let age: i32 = 25;        // 顯式類型
let height = 5.6;         // 類型推斷

// 不能重新賦值不可變變量
// age = 26;  // 錯誤:不能兩次賦值給不可變變量

// 使用 mut 聲明可變變量
let mut age = 25;
age = 26;  // 正確

// 不能改變類型
// age = "twenty-six";  // 錯誤:類型不匹配
```
</UniversalEditor>

### 為什麼默認不可變?

Rust 的默認不可變性可以防止 bug:
- **更清晰的代碼**:你知道變量不會改變
- **編譯器優化**:更容易優化
- **併發性**:在線程間更安全地共享

<UniversalEditor compare={true} title="實踐中的可變性">
```python !! py
# Python:一切都可以改變
def process(items):
    result = []
    for item in items:
        result.append(item * 2)
    return result

data = [1, 2, 3]
processed = process(data)
print(data)  # 仍然是 [1, 2, 3] - 未修改
```

```rust !! rs
// Rust:對變化的顯式控制
fn process(items: &[i32]) -> Vec<i32> {
    items.iter().map(|x| x * 2).collect()
}

let data = vec![1, 2, 3];
let processed = process(&data);
println!("{:?}", data);  // 仍然是 [1, 2, 3] - 有保證!
```
</UniversalEditor>

## 基本數據類型

Rust 有靜態類型系統,但編譯器擅長類型推斷。

### 整數類型

<UniversalEditor compare={true} title="整數類型">
```python !! py
# Python:任意精度整數
x = 42
x = 2**100  # 可以工作!Python 處理大整數
```

```rust !! rs
// Rust:固定大小整數
let x: i32 = 42;           // 32位有符號
let y: u64 = 100;          // 64位無符號
let z: i8 = -5;            // 8位有符號

// 整數字面量
let decimal = 98_222;      // 下劃線用於可讀性
let hex = 0xff;            // 前綴 0x
let octal = 0o77;          // 前綴 0o
let binary = 0b1111_0000;  // 前綴 0b
let byte = b'A';           // 僅 u8
```
</UniversalEditor>

**Rust 中的整數類型:**
- `i8, i16, i32, i64, i128, isize` (有符號)
- `u8, u16, u32, u64, u128, usize` (無符號)
- 默認是 `i32`
- `isize`/`usize` 是指針大小的

### 類型推斷

<UniversalEditor compare={true} title="類型推斷">
```python !! py

x = "hello" # 現在是 str - 類型可以改變
```


## 函數

函數是你將看到更顯著差異的地方。

<UniversalEditor compare={true} title="函數定義">
```python !! py
# Python:動態類型,不需要返回類型註解
def add(a, b):
    return a + b

# 帶類型提示(可選)
def add_with_hints(a: int, b: int) -> int:
    return a + b

# 可以返回不同類型
def flexible(x):
    if x > 0:
        return 42
    return "negative"
```

```rust !! rs
// Rust:靜態類型,顯式返回類型
fn add(a: i32, b: i32) -> i32 {
    a + b  // 沒有分號 = 返回值
}

// 必須聲明返回類型
// fn add(a: i32, b: i32) {  // 錯誤:缺少返回類型
//     a + b
// }

// 不能返回不同類型
// fn flexible(x: i32) -> ??? {  // 什麼類型?
//     if x > 0 {
//         return 42;
//     }
//     return "negative";  // 錯誤:類型不匹配
// }
```
</UniversalEditor>

### 語句 vs 表達式

這是一個關鍵的 Rust 概念!

<UniversalEditor compare={true} title="語句 vs 表達式">
```python !! py
# Python:一切都是語句(返回 None)
x = 5  # 語句
y = (x := 10)  # 賦值表達式(Python 3.8+)

# if 是語句
if x > 0:
    result = "positive"
else:
    result = "negative"
```

```rust !! rs
// Rust:語句不返回值,表達式返回值
let x = 5;  // 語句(注意分號)

// 表達式返回值
let y = {
    let x = 3;
    x + 1  // 表達式(沒有分號)
};  // y = 4

// if 是表達式!
let result = if x > 0 {
    "positive"
} else {
    "negative"
};  // 不需要三元運算符
```
</UniversalEditor>

**關鍵規則**:在 Rust 中,表達式結束沒有分號,語句結束有分號。

<UniversalEditor compare={true} title="表達式示例">
```python !! py
# Python:語句和表達式混合
def calculate(x):
    if x > 0:
        return x * 2
    return x * 3

result = calculate(5)
```

```rust !! rs
// Rust:一切都可以是表達式
fn calculate(x: i32) -> i32 {
    if x > 0 {
        x * 2  // 表達式值被返回
    } else {
        x * 3
    }  // 塊沒有分號
}

// 可以使用 match 使代碼更清晰
fn calculate_match(x: i32) -> i32 {
    match x.cmp(&0) {
        std::cmp::Ordering::Greater => x * 2,
        _ => x * 3,
    }
}
```
</UniversalEditor>

### 返回值

<UniversalEditor compare={true} title="返回值">
```python !! py
# Python:顯式 return 或隱式 None
def add(a, b):
    return a + b

def no_return():
    x = 5
    # 隱式返回 None
```

```rust !! rs
// Rust:最後一個表達式是返回值
fn add(a: i32, b: i32) -> i32 {
    a + b  // 沒有分號 = 返回這個
}

// 可以使用 return 關鍵字(但不常見)
fn add_explicit(a: i32, b: i32) -> i32 {
    return a + b;  // 使用 return 時分號可以
}

// 沒有返回類型的函數返回 ()
fn no_return() {
    let x = 5;
    // 隱式返回單元類型 ()
}
```
</UniversalEditor>

## 運算符

大多數運算符是相似的,但有一些差異。

<UniversalEditor compare={true} title="算術運算符">
```python !! py
# Python
a = 10 + 5   # 15
b = 10 - 5   # 5
c = 10 * 5   # 50
d = 10 / 3   # 3.333...
e = 10 // 3  # 3(整除)
f = 10 % 3   # 1
g = 2 ** 3   # 8(冪運算)
```

```rust !! rs
// Rust
let a = 10 + 5;   // 15
let b = 10 - 5;   // 5
let c = 10 * 5;   // 50
let d = 10 / 3;   // 3(整數除法!)
let e = 10 % 3;   // 1
let f = 2_i32.pow(3);  // 8(方法,不是運算符)

// 對於浮點數
let d = 10.0 / 3.0;  // 3.333...
```
</UniversalEditor>

<UniversalEditor compare={true} title="比較運算符">
```python !! py
# Python
a = 5
b = 10

a == b  # False
a != b  # True
a < b   # True
a <= b  # True
a > b   # False
a >= b  # False
```

```rust !! rs
// Rust(與 Python 相同)
let a = 5;
let b = 10;

a == b  // False
a != b  // True
a < b   // True
a <= b  // True
a > b   // False
a >= b  // False
```
</UniversalEditor>

<UniversalEditor compare={true} title="邏輯運算符">
```python !! py
# Python:單詞,不是符號
a = True
b = False

a and b   # False
a or b    # True
not a     # False
```

```rust !! rs
// Rust:符號,不是單詞
let a: bool = true;
let b: bool = false;

a && b   // False (AND)
a || b   // True (OR)
!a       // False (NOT)
```
</UniversalEditor>

## 字符串類型

Rust 有多種字符串類型 - 這是一個常見的困惑源!

<UniversalEditor compare={true} title="字符串類型">
```python !! py
# Python:只有 str
s1 = "Hello"
s2 = "World"
s3 = s1 + s2  # "HelloWorld"
```

```rust !! rs
// Rust:&str(字符串切片)和 String(擁有的)
let s1: &str = "Hello";  // &str - 不可變,固定大小
let s2: String = String::from("World");  // String - 可增長

let s3: String = s1.to_string() + &s2;  // "HelloWorld"

// 更多關於字符串的內容在模塊 7!
```
</UniversalEditor>

**快速總結:**
- `&str`:字符串切片,不可變,借用
- `String`:堆分配,可變,擁有

## 遮蔽

Rust 允許遮蔽 - 用相同名稱聲明新變量。

<UniversalEditor compare={true} title="遮蔽">
```python !! py
# Python:沒有遮蔽,只是重新賦值
x = 5
x = "hello"  # 同一個變量,不同類型
```

```rust !! rs
// Rust:遮蔽創建新變量
let x = 5;
let x = x + 1;  // 遮蔽 x
let x = "hello";  // 用不同類型遮蔽!

// 這與 mut 不同
let mut x = 5;
x = x + 1;  // 同一個變量,新值
// x = "hello";  // 錯誤:類型不匹配
```
</UniversalEditor>

**何時使用遮蔽:**
- 通過步驟轉換值
- 改變類型
- 保持名稱有意義

## 綜合實例

讓我們寫一個更復雜的例子:

<UniversalEditor compare={true} title="完整示例">
```python !! py
def calculate_stats(numbers):
    """計算總和和平均值。"""
    if not numbers:
        return 0, 0.0

    total = sum(numbers)
    average = total / len(numbers)

    return total, average

def main():
    data = [1, 2, 3, 4, 5]
    total, avg = calculate_stats(data)

    print(f"Total: {total}")
    print(f"Average: {avg:.2}")

if __name__ == "__main__":
    main()
```

```rust !! rs
fn calculate_stats(numbers: &[i32]) -> (i32, f64) {
    if numbers.is_empty() {
        return (0, 0.0);
    }

    let total: i32 = numbers.iter().sum();
    let average: f64 = total as f64 / numbers.len() as f64;

    (total, average)
}

fn main() {
    let data = vec![1, 2, 3, 4, 5];
    let (total, avg) = calculate_stats(&data);

    println!("Total: {}", total);
    println!("Average: {:.2}", avg);
}
```
</UniversalEditor>

## 總結

在本模塊中,你學習了:
- ✅ 變量默認不可變(`let` vs `let mut`)
- ✅ Rust 有帶推斷的靜態類型
- ✅ 函數需要顯式類型
- ✅ 表達式(無分號)vs 語句(有分號)
- ✅ `println!` 宏用於輸出
- ✅ 塊級作用域
- ✅ 用於轉換值的遮蔽

## 與 Python 的主要區別

1. **不可變性**:Rust 默認為不可變變量
2. **靜態類型**:類型在編譯時檢查
3. **顯式返回**:最後一個表達式被返回,不是 `return` 關鍵字
4. **塊作用域**:變量限制在其塊中
5. **沒有真值/假值**:必須使用顯式布爾條件
6. **字符串類型**:多種字符串類型(`&str` vs `String`)

## 練習

### 練習 1:溫度轉換

編寫一個將攝氏度轉換為華氏度的函數:
- 公式:`F = C * 9/5 + 32`
- 使用適當的類型和註解
- 打印結果,保留 1 位小數

<details>
<summary>解決方案</summary>

```rust
fn celsius_to_fahrenheit(celsius: f64) -> f64 {
    celsius * 9.0 / 5.0 + 32.0
}

fn main() {
    let celsius = 25.0;
    let fahrenheit = celsius_to_fahrenheit(celsius);
    println!("{:.1}°C = {:.1}°F", celsius, fahrenheit);
}
```

</details>

### 練習 2:圓的屬性

計算圓的面積和周長:
- 使用 `f64` 進行計算
- 返回元組 `(area, circumference)`
- π = 3.14159

<details>
<summary>解決方案</summary>

```rust
fn circle_properties(radius: f64) -> (f64, f64) {
    let pi = 3.14159;
    let area = pi * radius * radius;
    let circumference = 2.0 * pi * radius;
    (area, circumference)
}

fn main() {
    let radius = 5.0;
    let (area, circumference) = circle_properties(radius);
    println!("Radius: {}", radius);
    println!("Area: {:.2}", area);
    println!("Circumference: {:.2}", circumference);
}
```

</details>
