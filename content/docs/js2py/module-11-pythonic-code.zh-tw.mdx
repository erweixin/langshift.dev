## 1. 引言

每種編程語言都有其獨特的風格和哲學。“Pythonic”這個詞用來描述那些遵循 Python 社區公認的最佳實踐、充分利用語言特性、代碼簡潔易讀且高效的代碼風格。

對於從 JavaScript 過來的開發者來說，僅僅掌握語法是不夠的。學習編寫 Pythonic 代碼，意味著要開始用“Python 的方式”去思考和解決問題。

> 💡 **學習策略**：將編寫 Pythonic 代碼看作是學習一門語言的“習語”和“俚語”。它能讓你的代碼更受社區歡迎，也更易於他人理解和維護。

## 2. 列表推導式 (List Comprehensions)

列表推導式是 Python 中最具代表性的特性之一。它提供了一種簡潔優雅的方式來根據現有列表創建新列表，通常比使用 `map()` 和 `filter()` 更具可讀性。

<PythonEditor title="列表推導式 vs. map/filter" compare={true}>
```javascript !! js
// JavaScript (map and filter)
const numbers = [1, 2, 3, 4, 5];

// 獲取所有數字的平方
const squares = numbers.map(n => n * n);

// 獲取所有偶數的平方
const evenSquares = numbers
  .filter(n => n % 2 === 0)
  .map(n => n * n);
console.log(numbers);
console.log(squares);
console.log(evenSquares);
```

```python !! py
# Python (List Comprehensions)
numbers = [1, 2, 3, 4, 5]

# 獲取所有數字的平方 (Pythonic)
squares = [n * n for n in numbers]

# 獲取所有偶數的平方 (Pythonic)
even_squares = [n * n for n in numbers if n % 2 == 0]

# 非 Pythonic 的方式
even_squares_long = list(map(lambda n: n * n, filter(lambda n: n % 2 == 0, numbers)))
print(numbers)
print(squares)
print(even_squares)
print(even_squares_long)
```
</PythonEditor>

## 3. EAFP vs LBYL

這是兩種不同的編程風格，Python 社區普遍推崇 EAFP。

*   **LBYL (Look Before You Leap)**: “三思而後行”。在執行操作前，先進行各種檢查。這在 JavaScript 中很常見。
*   **EAFP (Easier to Ask for Forgiveness than Permission)**: “先斬後奏”。直接嘗試執行操作，如果出現問題（異常），再進行處理。

<PythonEditor title="EAFP vs. LBYL 對比" compare={true}>
```javascript !! js
// JavaScript (LBYL 風格)
const user = { profile: { name: 'Alice' } };

if (user && user.profile && user.profile.name) {
  console.log(user.profile.name);
} else {
  console.log('Name not found');
}

// 使用可選鏈操作符 ?. 更簡潔
console.log(user?.profile?.name || 'Name not found');
```

```python !! py
# Python (EAFP 風格 - Pythonic)
user = {"profile": {"name": "Alice"}}

try:
    print(user["profile"]["name"])
except (KeyError, TypeError):
    print("Name not found")

# Python (LBYL 風格 - 不那麼 Pythonic)
if "profile" in user and isinstance(user["profile"], dict) and "name" in user["profile"]:
    print(user["profile"]["name"])
else:
    print("Name not found")
```
</PythonEditor>

## 4. 使用 `enumerate` 進行迭代

當你在循環中同時需要索引和值時，不要使用 `range(len(my_list))`。Pythonic 的方式是使用內置函數 `enumerate`。

<PythonEditor title="enumerate vs. range(len(...))" compare={true}>
```javascript !! js
// JavaScript
const items = ['a', 'b', 'c'];

items.forEach((item, index) => {
  console.log(`Index: ${index}, Item: ${item}`);
});
```

```python !! py
# Python (Pythonic)
items = ['a', 'b', 'c']

for index, item in enumerate(items):
    print(f"Index: {index}, Item: {item}")

# Python (不那麼 Pythonic)
for i in range(len(items)):
    print(f"Index: {i}, Item: {items[i]}")
```
</PythonEditor>

## 5. 解包 (Unpacking)

Python 允許你將一個可迭代對象（如列表或元組）的元素“解包”到多個變量中。這使得代碼更加簡潔。

<PythonEditor title="解包 vs. 手動賦值" compare={true}>
```javascript !! js
// JavaScript (解構賦值)
const point = [10, 20, 30];
const [x, y, z] = point;

const [first, ...rest] = [1, 2, 3, 4];
// first = 1, rest = [2, 3, 4]

console.log(x, y, z);
console.log(first, rest);
```

```python !! py
# Python (解包)
point = (10, 20, 30)
x, y, z = point

# 星號表達式 (類似 JS 的 rest)
first, *rest, last = [1, 2, 3, 4, 5]
# first = 1, rest = [2, 3, 4], last = 5

print(x, y, z)
print(first, rest)
```
</PythonEditor>

## 6. 使用 `with` 語句管理資源

正如在“常見陷阱”模塊中提到的，使用 `with` 語句（上下文管理器）是處理需要顯式關閉的資源（如文件、數據庫連接）的最 Pythonic 的方式。它能確保資源即使在發生錯誤時也能被正確釋放。

<PythonEditor title="with 語句 vs. try...finally" compare={true} canRun={false}>
```javascript !! js
// JavaScript (沒有直接對應物，通常用 try...finally)
let resource;
try {
  resource = acquireResource();
  // use resource
} finally {
  if (resource) {
    releaseResource(resource);
  }
}
```

```python !! py
# Python (Pythonic)
with open("my_file.txt", "r") as f:
    content = f.read()
    # 文件會自動關閉，即使在讀取時發生錯誤

# Python (不那麼 Pythonic)
f = open("my_file.txt", "r")
try:
    content = f.read()
finally:
    f.close()
```
</PythonEditor>

## 7. 總結

編寫 Pythonic 代碼是一個持續的過程，它關乎於代碼的可讀性、簡潔性和效率。當你開始自然地使用列表推導式、`enumerate` 和 `with` 語句時，你就真正走上了成為一名地道 Python 開發者的道路。

**核心原則**：代碼首先是寫給人看的，其次才是給機器執行的。Pythonic 的代碼正是這一原則的最佳體現。
