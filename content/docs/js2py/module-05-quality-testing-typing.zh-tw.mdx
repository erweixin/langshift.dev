## 1. 引言

### 為什麼代碼質量、測試和類型提示很重要？

作為一名經驗豐富的 JavaScript 開發者，你一定深知 `ESLint`、`Prettier`、`Jest` 和 `TypeScript` 在大型項目中的重要性。它們是確保代碼一致性、可讀性和可維護性的基石。

Python 生態系統同樣擁有強大的工具集來解決這些問題。本模塊將向你介紹 Python 中的代碼質量、測試和類型提示工具，並與你熟悉的 JavaScript 工具進行對比。

**核心概念類比**

| Python 工具/概念 | JavaScript 工具/概念 | 主要功能 |
| --- | --- | --- |
| **Black** | `Prettier` | 自動化代碼格式化 |
| **Ruff** | `ESLint` | 代碼風格檢查、錯誤檢測 |
| **pytest** | `Jest` / `Mocha` | 單元測試、集成測試 |
| **Type Hints** | `TypeScript` | 靜態類型檢查 |
| **Mypy** | `tsc` (TypeScript Compiler) | 類型檢查器 |

> 💡 **學習策略**：將 Python 的這些工具視為你現有 JavaScript 工作流的“Python 版本”。它們的目標和理念是相似的，只是具體實現和語法有所不同。

## 2. 代碼質量與格式化

### 2.1 Black: 不妥協的代碼格式化工具

`Black` 是 Python 社區廣泛使用的代碼格式化工具，它以“不妥協”而聞名，意味著它只提供極少的配置選項。這確保了在任何項目中，代碼風格都能保持高度一致。

<PythonEditor title="代碼格式化對比：Black vs. Prettier" compare={true} canRun={false} height={800}>
```javascript !! js
// JavaScript (使用 Prettier 格式化前)
function longFunctionName(arg1, arg2, arg3, arg4, arg5, arg6) {
  console.log('這是一個很長的函數定義，Prettier 會自動換行');
  if (arg1 && arg2 && arg3 && arg4 && arg5 && arg6) {
    return { key1: 'value1', key2: 'value2', key3: 'value3' };
  }
}

// Prettier 格式化後
function longFunctionName(
  arg1,
  arg2,
  arg3,
  arg4,
  arg5,
  arg6,
) {
  console.log(
    '這是一個很長的函數定義，Prettier 會自動換行',
  );
  if (
    arg1 &&
    arg2 &&
    arg3 &&
    arg4 &&
    arg5 &&
    arg6
  ) {
    return {
      key1: 'value1',
      key2: 'value2',
      key3: 'value3',
    };
  }
}
```

```python !! py
# Python (使用 Black 格式化前)
def long_function_name(arg1, arg2, arg3, arg4, arg5, arg6):
  print('這是一個很長的函數定義，Black 會自動換行')
  if arg1 and arg2 and arg3 and arg4 and arg5 and arg6:
    return {'key1': 'value1', 'key2': 'value2', 'key3': 'value3'}

# Black 格式化後
def long_function_name(
    arg1,
    arg2,
    arg3,
    arg4,
    arg5,
    arg6,
):
    print("這是一個很長的函數定義，Black 會自動換行")
    if (
        arg1
        and arg2
        and arg3
        and arg4
        and arg5
        and arg6
    ):
        return {
            "key1": "value1",
            "key2": "value2",
            "key3": "value3",
        }
```
</PythonEditor>

### 2.2 Ruff: 高性能的代碼檢查器

`Ruff` 是一個用 Rust 編寫的 Python Linter，速度極快，可以替代 `Flake8`、`isort` 等多個工具。它能幫助你發現代碼中的潛在問題、改進代碼風格，並自動修復許多常見錯誤。

<PythonEditor title="代碼檢查對比：Ruff vs. ESLint" compare={true} canRun={false}>
```javascript !! js
// JavaScript (ESLint 可能會報告 'no-unused-vars')
import React from 'react'; // 'React' is defined but never used.

function MyComponent() {
  const unusedVar = 42; // 'unusedVar' is assigned a value but never used.
  return <div>Hello</div>;
}

// ESLint 還可以檢查規則，如強制使用 ===
if (a == 1) { // '==' is not allowed, use '==='
  // ...
}
```

```python !! py
# Python (Ruff 可能會報告 'F841' 和 'F401')
import os  # 'os' imported but unused (F401)
import sys

def my_function():
    unused_var = 42  # local variable 'unused_var' is assigned to but never used (F841)
    print("Hello from my_function")

# Ruff 還可以檢查許多其他問題，例如不必要的列表推導
# Instead of list(my_generator), use [...my_generator]
```
</PythonEditor>

## 3. 單元測試與 Pytest

`pytest` 是 Python 中最流行、功能最強大的測試框架。它以其簡潔的語法、強大的斷言功能和豐富的插件生態系統而著稱。

### 3.1 基本測試用例

與 `Jest` 類似，`pytest` 允許你使用簡單的函數來編寫測試用例，而不需要複雜的類結構。

<PythonEditor title="基本測試對比：pytest vs. Jest" compare={true} canRun={false}>
```javascript !! js
// JavaScript (Jest)
// a_plus_b.js
const a_plus_b = (a, b) => a + b;

// a_plus_b.test.js
const a_plus_b = require('./a_plus_b');

describe('a_plus_b', () => {
  it('should return the sum of two numbers', () => {
    expect(a_plus_b(1, 2)).toBe(3);
  });

  it('should handle negative numbers', () => {
    expect(a_plus_b(-1, -1)).toBe(-2);
  });
});
```

```python !! py
# Python (pytest)
# a_plus_b.py
def a_plus_b(a, b):
    return a + b

# test_a_plus_b.py
from a_plus_b import a_plus_b

def test_a_plus_b():
    assert a_plus_b(1, 2) == 3

def test_a_plus_b_negative():
    assert a_plus_b(-1, -1) == -2
```
</PythonEditor>

### 3.2 參數化測試

`pytest` 的參數化測試功能非常強大，可以讓你用不同的輸入數據多次運行同一個測試函數，這在 `Jest` 中通常通過 `test.each` 實現。

<PythonEditor title="參數化測試對比：pytest vs. Jest" compare={true} canRun={false}>
```javascript !! js
// JavaScript (Jest)
// a_plus_b.test.js
const a_plus_b = require('./a_plus_b');

describe('a_plus_b', () => {
  test.each([
    [1, 2, 3],
    [-1, 1, 0],
    [0, 0, 0],
    [100, 200, 300],
  ])('a_plus_b(%i, %i) should return %i', (a, b, expected) => {
    expect(a_plus_b(a, b)).toBe(expected);
  });
});
```

```python !! py
# Python (pytest)
# test_a_plus_b.py
import pytest
from a_plus_b import a_plus_b

@pytest.mark.parametrize("a, b, expected", [
    (1, 2, 3),
    (-1, 1, 0),
    (0, 0, 0),
    (100, 200, 300),
])
def test_a_plus_b_parameterized(a, b, expected):
    assert a_plus_b(a, b) == expected
```
</PythonEditor>

### 3.3 Fixtures: 管理測試依賴

`pytest` 的 `fixtures` 是一個核心特性，用於管理測試的依賴和狀態。它們類似於 `Jest` 中的 `beforeEach` 和 `afterEach`，但更加靈活和強大。

<PythonEditor title="Fixtures 對比：pytest vs. Jest" compare={true} canRun={false}>
```javascript !! js
// JavaScript (Jest)
// db.js
class Database {
  constructor() { this.users = []; }
  connect() { /* ... */ }
  disconnect() { /* ... */ }
  addUser(user) { this.users.push(user); }
}

// db.test.js
const Database = require('./db');

describe('Database', () => {
  let db;

  beforeEach(() => {
    db = new Database();
    db.connect();
  });

  afterEach(() => {
    db.disconnect();
  });

  it('should add a user', () => {
    db.addUser({ name: 'Alice' });
    expect(db.users).toHaveLength(1);
  });
});
```

```python !! py
# Python (pytest)
# db.py
class Database:
    def __init__(self):
        self.users = []
    def connect(self): ...
    def disconnect(self): ...
    def add_user(self, user):
        self.users.append(user)

# test_db.py
import pytest

@pytest.fixture
def db():
    database = Database()
    database.connect()
    yield database
    database.disconnect()

def test_add_user(db):
    db.add_user({"name": "Alice"})
    assert len(db.users) == 1
```
</PythonEditor>

## 4. 類型提示與靜態分析

Python 從 3.5 版本開始引入了類型提示（Type Hints），允許開發者為變量、函數參數和返回值添加類型信息。這與 `TypeScript` 的目標非常相似：在不改變運行時行為的情況下，提供靜態類型檢查，以提高代碼的健壯性和可讀性。

### 4.1 基本類型提示

<PythonEditor title="基本類型提示對比：Python vs. TypeScript" compare={true} canRun={false}>
```typescript !! ts
// TypeScript
function greet(name: string, age: number): string {
  return `Hello, ${name}. You are ${age} years old.`;
}

let user: { name: string; id: number } = { name: "Alice", id: 1 };
```

```python !! py
# Python
def greet(name: str, age: int) -> str:
    return f"Hello, {name}. You are {age} years old."

user: dict[str, int | str] = {"name": "Alice", "id": 1}
```
</PythonEditor>

### 4.2 複雜類型與泛型

和 `TypeScript` 一樣，Python 的類型系統也支持更復雜的類型，如列表、字典、泛型和自定義類型。

<PythonEditor title="複雜類型對比：Python vs. TypeScript" compare={true} canRun={false}>
```typescript !! ts
// TypeScript
type User = {
  name: string;
  id: number;
};

function processUsers(users: User[]): number[] {
  return users.map(user => user.id);
}

function getFirst<T>(items: T[]): T | undefined {
  return items[0];
}
```

```python !! py
# Python
from typing import List, TypeVar, Optional, TypedDict

# 使用 TypedDict 更精確地定義字典結構
class User(TypedDict):
    name: str
    id: int

def process_users(users: List[User]) -> List[int]:
    return [user["id"] for user in users]

T = TypeVar('T')

def get_first(items: List[T]) -> Optional[T]:
    return items[0] if items else None
```
</PythonEditor>

### 4.3 Mypy: Python 的靜態類型檢查器

`Mypy` 是 Python 的靜態類型檢查器，它讀取帶類型提示的 Python 代碼，分析並報告類型不一致的錯誤。它的作用類似於 `TypeScript` 的編譯器 `tsc`。

<PythonEditor title="類型檢查對比：Mypy vs. tsc" compare={true} canRun={false}>
```typescript !! ts
// TypeScript (tsc會報錯)
function add(a: number, b: number): number {
  return a + b;
}

add(1, "2"); // Error: Argument of type 'string' is not assignable to parameter of type 'number'.
```

```python !! py
# Python (Mypy會報錯)
def add(a: int, b: int) -> int:
    return a + b

add(1, "2")  # Error: Argument 2 to "add" has incompatible type "str"; expected "int"
```
</PythonEditor>
