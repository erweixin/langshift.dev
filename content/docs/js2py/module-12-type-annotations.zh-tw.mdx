## 1. 引言

### 為什麼需要類型註解？

作為一名 TypeScript 開發者，你一定深知類型系統在大型項目中的重要性。類型註解不僅能幫助 IDE 提供更好的智能提示，還能在編譯時捕獲潛在的錯誤，提高代碼的可維護性。

Python 從 3.5 版本開始引入類型提示（Type Hints），並在後續版本中不斷完善。雖然 Python 是動態類型語言，但類型註解為我們提供了類似 TypeScript 的開發體驗。

**核心概念映射**

| TypeScript 概念 | Python 概念 | 說明 |
| --- | --- | --- |
| `string`, `number`, `boolean` | `str`, `int`, `bool` | 基本類型 |
| `string[]`, `Array<string>` | `List[str]`, `list[str]` | 列表類型 |
| `{key: string}` | `Dict[str, Any]`, `dict[str, Any]` | 字典類型 |
| `interface` | `TypedDict`, `Protocol` | 接口定義 |
| `type` | `TypeAlias` | 類型別名 |
| `generic<T>` | `TypeVar('T')` | 泛型 |
| `union` | `Union`, `|` | 聯合類型 |
| `optional` | `Optional`, `| None` | 可選類型 |

> 💡 **學習策略**：將 Python 的類型註解視為 TypeScript 的"Python 版本"。雖然語法不同，但核心概念和思維方式是一致的。

## 2. 基本類型註解

### 2.1 變量類型註解

在 TypeScript 中，我們習慣為變量聲明類型。Python 也支持類似的語法。

let name: string = "Alice";
let age: number = 25;
let isActive: boolean = true;
let scores: number[] = [85, 92, 78];
let user: { name: string; age: number } = { name: "Bob", age: 30 };

// 使用 const 聲明常量
const PI: number = 3.14159;
const API_URL: string = "https://api.example.com";

from typing import List, Dict, Any

### 2.2 函數類型註解

函數是類型註解最重要的應用場景之一。Python 的函數類型註解語法與 TypeScript 非常相似。

<PythonEditor title="函數類型註解對比" compare={true} canRun={false}>
```typescript !! ts
// TypeScript
function greet(name: string, age: number): string {
  return `Hello, ${name}. You are ${age} years old.`;
}

function calculateArea(width: number, height: number): number {
  return width * height;
}

function getUserInfo(id: number): { name: string; email: string } | null {
  // 模擬從數據庫獲取用戶信息
  if (id === 1) {
    return { name: "Alice", email: "alice@example.com" };
  }
  return null;
}

// 箭頭函數
const multiply = (a: number, b: number): number => a * b;
```

```python !! py
# Python
from typing import Optional, Dict, Any

def greet(name: str, age: int) -> str:
    return f"Hello, {name}. You are {age} years old."

def calculate_area(width: float, height: float) -> float:
    return width * height

def get_user_info(user_id: int) -> Optional[dict[str, str]]:
    # 模擬從數據庫獲取用戶信息
    if user_id == 1:
        return {"name": "Alice", "email": "alice@example.com"}
    return None

# Lambda 函數（Python 的箭頭函數）
multiply = lambda a: float, b: float: a * b
```
</PythonEditor>

### 3.3 聯合類型和可選類型

Python 的聯合類型對應 TypeScript 的 union 類型，用於表示一個值可能是多種類型中的一種。

<PythonEditor title="聯合類型對比" compare={true} canRun={false}>
```typescript !! ts
// TypeScript
type Status = "pending" | "success" | "error";
type ID = string | number;
type Result<T> = T | null | undefined;

function processStatus(status: Status): string {
  switch (status) {
    case "pending": return "處理中...";
    case "success": return "成功";
    case "error": return "錯誤";
  }
}

function findUser(id: ID): User | null {
  // 根據 ID 查找用戶
  return null;
}

// 可選參數
function greet(name: string, title?: string): string {
  return title ? `Hello, ${title} ${name}` : `Hello, ${name}`;
}
```

```python !! py
# Python
from typing import Union, Optional, Literal

# 字面量聯合類型 (Python 3.8+)
Status = Literal["pending", "success", "error"]
ID = Union[str, int]  # 或者使用 str | int (Python 3.10+)
Result = Union[str, None]  # 或者使用 str | None

def process_status(status: Status) -> str:
    if status == "pending":
        return "處理中..."
    elif status == "success":
        return "成功"
    elif status == "error":
        return "錯誤"
    else:
        raise ValueError(f"Unknown status: {status}")

def find_user(user_id: ID) -> Optional[dict[str, Any]]:
    # 根據 ID 查找用戶
    return None

# 可選參數
def greet(name: str, title: Optional[str] = None) -> str:
    if title:
        return f"Hello, {title} {name}"
    return f"Hello, {name}"
```
</PythonEditor>

## 4. 高級類型註解

### 4.1 泛型

Python 的泛型系統與 TypeScript 的泛型概念非常相似，都允許創建可重用的類型模板。

<PythonEditor title="泛型對比" compare={true} canRun={false}>
```typescript !! ts
// TypeScript
interface Box<T> {
  value: T;
  getValue(): T;
  setValue(value: T): void;
}

class NumberBox implements Box<number> {
  constructor(private _value: number) {}
  
  get value(): number { return this._value; }
  getValue(): number { return this._value; }
  setValue(value: number): void { this._value = value; }
}

// 泛型函數
function identity<T>(arg: T): T {
  return arg;
}

function first<T>(array: T[]): T | undefined {
  return array[0];
}

// 約束泛型
interface Lengthwise {
  length: number;
}

function logLength<T extends Lengthwise>(arg: T): T {
  console.log(arg.length);
  return arg;
}
```

```python !! py
# Python
from typing import TypeVar, Generic, Protocol

# 定義類型變量
T = TypeVar('T')
U = TypeVar('U')

# 泛型類
class Box(Generic[T]):
    def __init__(self, value: T):
        self._value = value
    
    @property
    def value(self) -> T:
        return self._value
    
    def get_value(self) -> T:
        return self._value
    
    def set_value(self, value: T) -> None:
        self._value = value

class NumberBox(Box[int]):
    pass

# 泛型函數
def identity(arg: T) -> T:
    return arg

def first(array: list[T]) -> T | None:
    return array[0] if array else None

# 約束泛型（使用 Protocol）
class Lengthwise(Protocol):
    def __len__(self) -> int: ...

def log_length(arg: T) -> T:
    print(len(arg))
    return arg
```
</PythonEditor>

### 4.2 類型別名和接口

Python 使用 `TypeAlias` 和 `Protocol` 來實現類似 TypeScript 的 `type` 和 `interface` 功能。

<PythonEditor title="類型別名和接口對比" compare={true} canRun={false}>
```typescript !! ts
// TypeScript
// 類型別名
type Point = {
  x: number;
  y: number;
};

type Coordinates = Point;
type UserID = string;
type Status = "active" | "inactive" | "pending";

// 接口
interface User {
  id: UserID;
  name: string;
  email: string;
  status: Status;
}

interface AdminUser extends User {
  permissions: string[];
  canDelete: boolean;
}

// 函數類型
type Handler = (event: string, data: any) => void;
type AsyncHandler = (event: string, data: any) => Promise<void>;
```

```python !! py
# Python
from typing import TypeAlias, Protocol, Callable, Any
from typing_extensions import TypedDict

# 類型別名 (Python 3.10+)
Point: TypeAlias = dict[str, float]
Coordinates: TypeAlias = Point
UserID: TypeAlias = str
Status: TypeAlias = Literal["active", "inactive", "pending"]

# 使用 TypedDict 定義接口
class User(TypedDict):
    id: UserID
    name: str
    email: str
    status: Status

class AdminUser(User):
    permissions: list[str]
    can_delete: bool

# 函數類型
Handler: TypeAlias = Callable[[str, Any], None]
AsyncHandler: TypeAlias = Callable[[str, Any], Any]  # 返回類型為 Any 因為 Python 沒有 Promise 類型
```
</PythonEditor>

### 4.3 回調函數和異步類型

Python 的異步編程模型與 TypeScript 有所不同，但類型註解的概念是相似的。

<PythonEditor title="回調函數和異步類型對比" compare={true} canRun={false}>
```typescript !! ts
// TypeScript
// 回調函數類型
type Callback<T> = (error: Error | null, result: T) => void;
type EventHandler = (event: Event) => void;

// 異步函數
async function fetchUser(id: number): Promise<User> {
  const response = await fetch(`/api/users/${id}`);
  return response.json();
}

// 異步回調
function processData(data: string[], callback: Callback<string[]>) {
  setTimeout(() => {
    const processed = data.map(item => item.toUpperCase());
    callback(null, processed);
  }, 1000);
}

// 事件處理
function handleClick(event: Event): void {
  console.log('Button clicked');
}
```

```python !! py
# Python
import asyncio
from typing import Callable, Any, Awaitable
from typing_extensions import TypedDict

# 回調函數類型
Callback = Callable[[Exception | None, Any], None]
EventHandler = Callable[[Any], None]

# 異步函數
async def fetch_user(user_id: int) -> dict[str, Any]:
    # 模擬異步請求
    await asyncio.sleep(0.1)
    return {"id": user_id, "name": "Alice", "email": "alice@example.com"}

# 異步回調（使用 asyncio）
async def process_data(data: list[str], callback: Callable[[list[str]], None]) -> None:
    await asyncio.sleep(1)
    processed = [item.upper() for item in data]
    callback(processed)

# 事件處理
def handle_click(event: Any) -> None:
    print("Button clicked")
```

## 5. 實際應用場景

### 5.1 API 響應類型定義

在實際項目中，我們經常需要為 API 響應定義類型。

<PythonEditor title="API 響應類型定義對比" compare={true} canRun={false}>
```typescript !! ts
// TypeScript
interface ApiResponse<T> {
  success: boolean;
  data: T;
  message: string;
  timestamp: string;
}

interface User {
  id: number;
  name: string;
  email: string;
  createdAt: string;
}

interface UserListResponse extends ApiResponse<User[]> {
  pagination: {
    page: number;
    limit: number;
    total: number;
  };
}

// 使用示例
async function fetchUsers(page: number = 1): Promise<UserListResponse> {
  const response = await fetch(`/api/users?page=${page}`);
  return response.json();
}

// 錯誤處理
interface ApiError {
  code: string;
  message: string;
  details?: any;
}
```

```python !! py
# Python
from typing import TypeVar, Generic
from typing_extensions import TypedDict
from datetime import datetime

# 定義泛型 API 響應
T = TypeVar('T')

class ApiResponse(TypedDict, Generic[T]):
    success: bool
    data: T
    message: str
    timestamp: str

class User(TypedDict):
    id: int
    name: str
    email: str
    created_at: str

class Pagination(TypedDict):
    page: int
    limit: int
    total: int

class UserListResponse(ApiResponse[list[User]]):
    pagination: Pagination

# 使用示例
async def fetch_users(page: int = 1) -> UserListResponse:
    # 模擬異步請求
    await asyncio.sleep(0.1)
    return {
        "success": True,
        "data": [
            {"id": 1, "name": "Alice", "email": "alice@example.com", "created_at": "2024-01-01"}
        ],
        "message": "Users fetched successfully",
        "timestamp": datetime.now().isoformat(),
        "pagination": {"page": page, "limit": 10, "total": 100}
    }

# 錯誤處理
class ApiError(TypedDict):
    code: str
    message: str
    details: dict[str, Any] | None
```
</PythonEditor>

### 5.2 配置管理

類型註解在配置管理中特別有用，可以確保配置對象的類型安全。

<PythonEditor title="配置管理類型定義對比" compare={true}>
```typescript !! ts
// TypeScript
interface DatabaseConfig {
  host: string;
  port: number;
  username: string;
  password: string;
  database: string;
  ssl: boolean;
}

interface RedisConfig {
  host: string;
  port: number;
  password?: string;
  db: number;
}

interface AppConfig {
  env: 'development' | 'production' | 'test';
  port: number;
  database: DatabaseConfig;
  redis: RedisConfig;
  cors: {
    origin: string[];
    credentials: boolean;
  };
}

// 配置驗證函數
function validateConfig(config: any): config is AppConfig {
  return (
    typeof config.env === 'string' &&
    typeof config.port === 'number' &&
    config.database &&
    config.redis
  );
}
```

```python !! py
# Python
from typing import Literal
from typing_extensions import TypedDict

class DatabaseConfig(TypedDict):
    host: str
    port: int
    username: str
    password: str
    database: str
    ssl: bool

class RedisConfig(TypedDict, total=False):
    host: str
    port: int
    password: str | None
    db: int

class CorsConfig(TypedDict):
    origin: list[str]
    credentials: bool

class AppConfig(TypedDict):
    env: Literal["development", "production", "test"]
    port: int
    database: DatabaseConfig
    redis: RedisConfig
    cors: CorsConfig

# 配置驗證函數
def validate_config(config: dict[str, Any]) -> bool:
    required_fields = ["env", "port", "database", "redis", "cors"]
    return all(field in config for field in required_fields)

# 類型安全的配置獲取
def get_database_config(config: AppConfig) -> DatabaseConfig:
    return config["database"]
```

### 6.2 常見類型錯誤和解決方案

<PythonEditor title="常見類型錯誤示例" compare={true}>
```typescript !! ts
// TypeScript 常見錯誤
function processUser(user: User) {
  // 錯誤：Property 'age' does not exist on type 'User'
  console.log(user.age);
  
  // 錯誤：Type 'string' is not assignable to type 'number'
  const id: number = "123";
  
  // 錯誤：Argument of type 'string' is not assignable to parameter of type 'number'
  function add(a: number, b: number): number {
    return a + b;
  }
  add("1", 2);
}
```

```python !! py
# Python 常見類型錯誤
from typing import TypedDict

class User(TypedDict):
    name: str
    email: str

def process_user(user: User) -> None:
    # 錯誤：TypedDict "User" has no key 'age'
    print(user["age"])  # mypy 會報錯
    
    # 錯誤：Incompatible types in assignment
    user_id: int = "123"  # mypy 會報錯
    
    # 錯誤：Argument 1 to "add" has incompatible type "str"
    def add(a: int, b: int) -> int:
        return a + b
    
    add("1", 2)  # mypy 會報錯

# 解決方案：使用類型斷言或類型檢查
def safe_process_user(user: User) -> None:
    # 使用 get 方法避免 KeyError
    age = user.get("age", 0)
    print(f"Age: {age}")
    
    # 類型轉換
    user_id: int = int("123")
    
    # 運行時類型檢查
    def safe_add(a: Any, b: Any) -> int:
        if isinstance(a, int) and isinstance(b, int):
            return a + b
        raise TypeError("Both arguments must be integers")
```

## 7. 最佳實踐

### 7.1 漸進式類型註解

與 TypeScript 類似，Python 也支持漸進式類型註解，你可以逐步為代碼添加類型信息。

<PythonEditor title="漸進式類型註解示例" compare={true}>
```typescript !! ts
// TypeScript 漸進式類型註解
// 1. 從 any 開始
function processData(data: any): any {
  return data.map(item => item.toUpperCase());
}

// 2. 添加基本類型
function processData(data: any[]): string[] {
  return data.map(item => item.toUpperCase());
}

// 3. 完善類型定義
interface DataItem {
  name: string;
  value: number;
}

function processData(data: DataItem[]): string[] {
  return data.map(item => item.name.toUpperCase());
}
```

```python !! py
# Python 漸進式類型註解
from typing import Any, List

# 1. 從 Any 開始
def process_data(data: Any) -> Any:
    return [item.upper() for item in data]

# 2. 添加基本類型
def process_data(data: list[Any]) -> list[str]:
    return [item.upper() for item in data]

# 3. 完善類型定義
class DataItem(TypedDict):
    name: str
    value: int

def process_data(data: list[DataItem]) -> list[str]:
    return [item["name"].upper() for item in data]
```

## 8. 總結

恭喜你！現在你已經深入瞭解了 Python 類型註解系統與 TypeScript 的對應關係。

### 關鍵要點

1. **基本類型映射**：Python 的 `str`, `int`, `bool` 對應 TypeScript 的 `string`, `number`, `boolean`
2. **容器類型**：Python 的 `list[T]`, `dict[K, V]` 對應 TypeScript 的 `T[]`, `Record<K, V>`
3. **聯合類型**：Python 的 `Union[T, U]` 或 `T | U` 對應 TypeScript 的 `T | U`
4. **泛型**：Python 的 `TypeVar` 和 `Generic` 對應 TypeScript 的泛型語法
5. **接口**：Python 的 `TypedDict` 和 `Protocol` 對應 TypeScript 的 `interface`

### 實踐建議

1. **漸進式採用**：從關鍵函數開始添加類型註解，逐步擴展到整個項目
2. **使用 Mypy**：配置 Mypy 進行靜態類型檢查，在 CI/CD 中集成
3. **保持一致性**：建立團隊的類型註解規範，保持代碼風格一致
4. **文檔化**：使用類型註解作為代碼文檔，提高代碼可讀性
