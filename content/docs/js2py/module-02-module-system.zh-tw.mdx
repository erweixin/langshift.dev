## 1. 引言

### 為什麼需要模塊系統？

在前端開發中，我們已經習慣了使用 ES6 模塊或 CommonJS 來組織代碼。Python 也有自己的模塊系統，雖然語法不同，但核心概念是相似的。

**模塊化的核心價值**
- 代碼複用：避免重複編寫相同的功能
- 命名空間隔離：避免變量名衝突
- 項目組織：將相關功能組織在一起
- 依賴管理：明確項目依賴關係

> 💡 **學習策略**：將 Python 的模塊系統理解為 JavaScript 模塊系統的"方言版本"

## 2. Python 模塊系統基礎

### 2.1 什麼是模塊？

在 Python 中，任何 `.py` 文件都可以作為一個模塊。這類似於 JavaScript 中的單個 `.js` 文件。

<PythonEditor title="創建和使用模塊" compare={true} canRun={false}>
```javascript !! js
// math_utils.js - JavaScript 模塊
export function add(a, b) {
    return a + b;
}

export function multiply(a, b) {
    return a * b;
}

export const PI = 3.14159;

// 默認導出
export default function calculator() {
    console.log("計算器已初始化");
}
```

```python !! py
# math_utils.py - Python 模塊
def add(a, b):
    """加法函數"""
    return a + b

def multiply(a, b):
    """乘法函數"""
    return a * b

# 模塊級變量（類似 JavaScript 的常量）
PI = 3.14159

# Python 沒有默認導出的概念，但可以定義 __all__ 來控制 from module import * 的行為
__all__ = ['add', 'multiply', 'PI']
```
</PythonEditor>

擴展閱讀：[__all__](https://docs.python.org/3/tutorial/modules.html#importing-from-a-package)

### 2.2 導入模塊的方式

Python 提供了多種導入模塊的方式，每種都有其適用場景。

<PythonEditor title="模塊導入方式對比" compare={true} canRun={false}>
```javascript !! js
// JavaScript 導入方式

// 1. 命名導入
import { add, multiply } from './math_utils.js';

// 2. 默認導入
import calculator from './math_utils.js';

// 3. 命名空間導入
import * as MathUtils from './math_utils.js';

// 4. 重命名導入
import { add as addFunction } from './math_utils.js';

// 5. 混合導入
import calculator, { add, multiply } from './math_utils.js';

console.log(add(5, 3));        // 8
console.log(multiply(4, 2));   // 8
console.log(MathUtils.PI);     // 3.14159
```

```python !! py
# Python 導入方式

# 1. 導入整個模塊
import math_utils

# 2. 從模塊導入特定函數/變量
from math_utils import add, multiply, PI

# 3. 導入所有內容（不推薦）
from math_utils import *

# 4. 重命名導入
from math_utils import add as add_function

# 5. 重命名模塊
import math_utils as math

# 6. 從模塊導入並重命名
from math_utils import add as add_func, multiply as mul

print(add(5, 3))           // 8
print(multiply(4, 2))      // 8
print(PI)                  // 3.14159
print(math_utils.add(10, 5))  // 15
```
</PythonEditor>

**導入方式對比表**

| 功能 | JavaScript | Python | 說明 |
|------|------------|--------|------|
| **導入整個模塊** | `import * as Module` | `import module` | Python 更簡潔 |
| **命名導入** | `import { func }` | `from module import func` | 語法不同但概念相同 |
| **重命名** | `import { func as newName }` | `from module import func as newName` | 兩者都支持 |
| **默認導入** | `import defaultFunc` | 無對應概念 | Python 沒有默認導出 |
| **命名空間導入** | `import * as Namespace` | `import module as namespace` | 概念相同 |

### 2.4 運行模塊：腳本 vs 模塊

這是 Python 初學者最容易困惑的地方之一：如何正確運行一個 Python 文件？

**直接運行 vs 模塊運行**

| 運行方式 | 命令 | `sys.path` (搜索路徑) | `__name__` | 適用場景 |
|----------|------|-----------------------|------------|----------|
| **腳本運行** | `python file.py` | 包含 `file.py` 所在的目錄 | `"__main__"` | 簡單的單文件腳本 |
| **模塊運行** | `python -m package.module` | 包含**當前工作目錄** | `"__main__"` | 項目中的模塊，需要引用同級或上級包時 |

**常見錯誤示例**

假設目錄結構如下：
```
project/
├── main.py
└── utils.py
```

如果在 `main.py` 中有 `from . import utils` (相對導入)：
- 運行 `python project/main.py` 會**報錯**：`ImportError: attempted relative import with no known parent package`。
- 運行 `python -m project.main` (在 `project` 的上一級目錄) 會**成功**。

> 💡 **最佳實踐**：在開發項目時，習慣在項目根目錄使用 `python -m package.module` 的方式運行代碼，這樣可以確保 `sys.path` 正確，且相對導入能正常工作。

## 3. 包（Package）系統


### 3.1 什麼是包？

包是包含多個模塊的目錄，類似於 JavaScript 中的 npm 包或目錄結構。

<PythonEditor title="包結構示例" compare={true} canRun={false}>
```javascript !! js
// JavaScript 包結構示例
// my-package/
// ├── package.json
// ├── index.js
// ├── utils/
// │   ├── math.js
// │   └── string.js
// └── tests/
//     └── test.js

// index.js - 主入口文件
export { add, multiply } from './utils/math.js';
export { capitalize, reverse } from './utils/string.js';

// utils/math.js
export function add(a, b) {
    return a + b;
}

export function multiply(a, b) {
    return a * b;
}

// utils/string.js
export function capitalize(str) {
    return str.charAt(0).toUpperCase() + str.slice(1);
}

export function reverse(str) {
    return str.split('').reverse().join('');
}
```

```python !! py
    """加法函數"""
    return a + b

def multiply(a, b):
    """乘法函數"""
    return a * b

# string_utils.py

    """首字母大寫"""
    return text.capitalize()

def reverse(text):
    """字符串反轉"""
    return text[::-1]
```

### 5.1 創建一個簡單的計算器包

讓我們創建一個完整的計算器包，展示模塊系統的實際應用。

<PythonEditor title="計算器包示例" compare={true}>
```javascript !! js
// calculator.js - JavaScript 版本
class Calculator {
    constructor() {
        this.history = [];
    }
    
    add(a, b) {
        const result = a + b;
        this.history.push(`${a} + ${b} = ${result}`);
        return result;
    }
    
    subtract(a, b) {
        const result = a - b;
        this.history.push(`${a} - ${b} = ${result}`);
        return result;
    }
    
    multiply(a, b) {
        const result = a * b;
        this.history.push(`${a} * ${b} = ${result}`);
        return result;
    }
    
    divide(a, b) {
        if (b === 0) {
            throw new Error("除數不能為零");
        }
        const result = a / b;
        this.history.push(`${a} / ${b} = ${result}`);
        return result;
    }
    
    getHistory() {
        return this.history;
    }
    
    clearHistory() {
        this.history = [];
    }
}

// 工具函數
function validateNumber(num) {
    return typeof num === 'number' && !isNaN(num);
}

function formatResult(result) {
    return `結果: ${result}`;
}

const calc = new Calculator()
console.log(calc.add(10, 5))
console.log(calc.multiply(4, 3))
console.log("計算歷史:", calc.getHistory())
```

```python !! py
# calculator.py - Python 版本
class Calculator:
    """計算器類，支持基本運算和歷史記錄"""
    
    def __init__(self):
        self.history = []
    
    def add(self, a, b):
        """加法運算"""
        result = a + b
        self.history.append(f"{a} + {b} = {result}")
        return result
    
    def subtract(self, a, b):
        """減法運算"""
        result = a - b
        self.history.append(f"{a} - {b} = {result}")
        return result
    
    def multiply(self, a, b):
        """乘法運算"""
        result = a * b
        self.history.append(f"{a} * {b} = {result}")
        return result
    
    def divide(self, a, b):
        """除法運算"""
        if b == 0:
            raise ValueError("除數不能為零")
        result = a / b
        self.history.append(f"{a} / {b} = {result}")
        return result
    
    def get_history(self):
        """獲取計算歷史"""
        return self.history
    
    def clear_history(self):
        """清空計算歷史"""
        self.history = []

# 工具函數
def validate_number(num):
    """驗證數字"""
    return isinstance(num, (int, float)) and not isinstance(num, bool)

def format_result(result):
    """格式化結果"""
    return f"結果: {result}"

# 如果直接運行此文件
if __name__ == "__main__":
    calc = Calculator()
    print(calc.add(10, 5))
    print(calc.multiply(4, 3))
    print("計算歷史:", calc.get_history())
```

```python !! py
from calculator import Calculator, validate_number, format_result

def main():
    # 創建計算器實例
    calc = Calculator()
    
    # 基本運算
    print(format_result(calc.add(10, 5)))      # 結果: 15
    print(format_result(calc.subtract(10, 3))) # 結果: 7

    print(validate_number(3.14))   # True
```

### 6.2 動態導入

Python 支持動態導入模塊，類似於 JavaScript 的動態 import()。

<PythonEditor title="動態導入示例" compare={true} canRun={false} height={600}>
```javascript !! js
// JavaScript 動態導入
async function loadModule(moduleName) {
    try {
        const module = await import(`./modules/${moduleName}.js`);
        return module;
    } catch (error) {
        console.error(`無法加載模塊 ${moduleName}:`, error);
        return null;
    }
}

// 使用動態導入
async function main() {
    const mathModule = await loadModule('math');
    if (mathModule) {
        console.log(mathModule.add(5, 3));
    }
    
    const stringModule = await loadModule('string');
    if (stringModule) {
        console.log(stringModule.capitalize('hello'));
    }
}

main();
```

```python !! py
# Python 動態導入
import importlib
import sys

def load_module(module_name):
    """動態加載模塊"""
    try:
        module = importlib.import_module(f"modules.{module_name}")
        return module
    except ImportError as error:
        print(f"無法加載模塊 {module_name}: {error}")
        return None

# 使用動態導入
def main():
    math_module = load_module('math_utils')
    if math_module:
        print(math_module.add(5, 3))
    
    string_module = load_module('string_utils')
    if string_module:
        print(string_module.capitalize('hello'))

if __name__ == "__main__":
    main()
```

### 6.3 循環導入陷阱 (Circular Imports)

循環導入是指兩個或多個模塊相互導入對方，形成一個閉環。這在 Python 中會導致 `ImportError` 或 `AttributeError`，因為模塊在導入時會執行頂層代碼。

**問題示例**

```python
# a.py
from b import func_b

def func_a():
    func_b()

# b.py
from a import func_a

def func_b():
    func_a()
```

**解決方案**

1.  **重構代碼**：將共享的依賴提取到第三個模塊（推薦）。
2.  **延遲導入**：將導入語句移到函數/方法內部（僅在需要時導入）。

```python
# b.py (修復後)
def func_b():
    # 延遲導入，打破循環
    from a import func_a
    func_a()
```

## 8. 總結

### 關鍵概念回顧

1. **模塊**：任何 `.py` 文件都可以作為模塊使用
2. **包**：包含 `__init__.py` 的目錄，可以包含多個模塊
3. **導入方式**：`import`、`from ... import`、`as` 重命名
4. **特殊變量**：`__name__`、`__file__`、`__all__`
5. **項目組織**：合理的目錄結構和依賴管理

### JavaScript vs Python 對比總結

| 概念 | JavaScript | Python | 說明 |
|------|------------|--------|------|
| **模塊文件** | `.js` 文件 | `.py` 文件 | 都是單個文件作為模塊 |
| **包** | 目錄 + `package.json` | 目錄 + `__init__.py` | Python 需要 `__init__.py` |
| **導入語法** | `import { func }` | `from module import func` | 語法不同但概念相同 |
| **默認導出** | `export default` | 無對應概念 | Python 沒有默認導出 |
| **相對導入** | `../utils/math` | `..utils.math_utils` | 都支持相對路徑 |
| **動態導入** | `import()` | `importlib.import_module()` | 都支持運行時導入 |
