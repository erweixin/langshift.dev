# Kotlin 語言介紹

歡迎來到 JavaScript → Kotlin 學習之旅！本模組將幫助你了解 Kotlin 語言的設計哲學、生態系統，以及如何從 JavaScript 開發者的角度快速掌握這門現代程式語言。

## 什麼是 Kotlin？

Kotlin 是由 JetBrains 開發的一種現代、簡潔、安全的程式語言，運行在 Java 虛擬機器（JVM）上。它被 Google 選為 Android 開發的官方語言，同時支援伺服器端開發、Web 開發和跨平台應用開發。

### Kotlin 的核心特性

- **空安全**: 編譯時檢查空值，避免運行時空指標異常
- **協程**: 強大的非同步程式設計支援，比 JavaScript 的 Promise 更靈活
- **函數式程式設計**: 豐富的函數式程式設計特性
- **互操作性**: 與 Java 100% 互操作
- **跨平台**: Kotlin Multiplatform 支援多平台開發
- **簡潔語法**: 比 Java 更簡潔，比 JavaScript 更安全

## 語言對比範例

讓我們通過一個簡單的例子來感受兩種語言的差異：

<UniversalEditor title="Hello World 對比" compare={true}>
```javascript !! js
// JavaScript
let message = "Hello, World!";
console.log(message);

function greet(name) {
    return `Hello, ${name}!`;
}

const result = greet("Kotlin");
console.log(result);
```

```kotlin !! kt
// Kotlin
val message = "Hello, World!"
println(message)

fun greet(name: String): String {
    return "Hello, $name!"
}

val result = greet("Kotlin")
println(result)
```
</UniversalEditor>

### 主要差異

1. **變數宣告**: JavaScript 使用 `let/const`，Kotlin 使用 `val/var`
2. **型別註解**: Kotlin 需要顯式型別宣告
3. **字串模板**: Kotlin 使用 `$` 而不是 `${}`
4. **函數宣告**: Kotlin 使用 `fun` 關鍵字
5. **輸出函數**: Kotlin 使用 `println()` 而不是 `console.log()`

## Kotlin 生態系統

### JVM 平台
Kotlin 運行在 JVM 上，這意味著：
- 可以使用所有 Java 庫和框架
- 享受 JVM 的效能優化
- 跨平台相容性

### 開發工具
- **IntelliJ IDEA**: 官方 IDE，提供最佳開發體驗
- **Android Studio**: Android 開發專用 IDE
- **Gradle**: 現代建構系統
- **Kotlin Playground**: 線上程式碼編輯器

## 協程：Kotlin 的殺手鐧

協程是 Kotlin 最強大的特性之一，讓我們對比一下：

<UniversalEditor title="非同步程式設計對比" compare={true}>
```javascript !! js
// JavaScript Promise
async function fetchData() {
    try {
        const response = await fetch('https://api.example.com/data');
        const data = await response.json();
        return data;
    } catch (error) {
        console.error('Error:', error);
    }
}

fetchData().then(data => {
    console.log('Data:', data);
});
```

```kotlin !! kt
// Kotlin 協程
suspend fun fetchData(): String {
    return try {
        val response = URL("https://api.example.com/data").readText()
        response
    } catch (error: Exception) {
        println("Error: $error")
        ""
    }
}

// 啟動協程
runBlocking {
    val data = fetchData()
    println("Data: $data")
}
```
</UniversalEditor>

### 協程的優勢
- **更輕量**: 比執行緒更輕量，可以建立數千個協程
- **更簡單**: 程式碼看起來像同步程式碼，但實際是非同步執行
- **更靈活**: 支援取消、超時、異常處理等高級特性
