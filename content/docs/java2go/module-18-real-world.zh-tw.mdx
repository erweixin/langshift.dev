本模組探索 Go 中的真實世界專案架構模式，與 Java 企業應用程式進行比較。

## 專案結構

**Java 企業結構：**
- 分層架構（controller、service、repository）
- 針對不同關注點的單獨模組
- 大量使用依賴注入
- 配置驅動

**Go 專案結構：**
- 實用的、領域驅動的組織
- 基於介面的設計
- 最少的魔法，顯式的依賴
- 程式碼組織優於框架慣例

